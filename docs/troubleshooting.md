# トラブルシューティング

## ⚠️ 最も一般的な問題：Google Driveの共有設定

**この拡張機能の一番よくある問題は、Google Driveの共有設定に起因します。**

### 必要な共有設定

拡張機能が動作するためには、画像が以下の設定になっている必要があります：

- ✅ **「リンクを知っている全員が閲覧可」**
- ✅ **「一般公開」**
- ❌ **「制限付き」** (コピー不可)
- ❌ **特定ユーザーのみ共有** (コピー不可)

### 共有設定の変更方法

```
1. Google Driveで画像ファイルを右クリック
2. 「共有」をクリック
3. 「制限付き」をクリック
4. 「リンクを知っている全員」を選択
5. 「閲覧者」権限で保存
6. 「完了」をクリック
```

### 共有設定の確認方法

画像ファイルを選択した時に、ファイル名の下に表示されるアイコンで確認：
- 🔗 リンクアイコン: 「リンク共有」設定 ✅
- 🌍 地球アイコン: 「一般公開」設定 ✅
- 🔒 鍵アイコン: 「制限付き」設定 ❌

## よくある問題と解決方法

### 🚫 画像がコピーされない

#### 症状
- 右クリックメニューは表示されるが、実際にクリップボードにコピーされない
- 他のアプリケーションで貼り付けができない

#### 原因と対処法

**1. クリップボード権限がない**
```
解決方法:
1. chrome://settings/content/clipboard を開く
2. Google Drive (https://drive.google.com) に「許可」を設定
3. 拡張機能を再読み込み
```

**2. ブラウザがClipboard APIに対応していない**
```
対応状況:
✅ Chrome 66+
✅ Edge 79+
✅ Firefox 87+
❌ Safari (一部制限あり)

解決方法: 対応ブラウザを使用
```

**3. Google Driveの認証問題**
```
解決方法:
1. Google Driveからログアウト
2. 再度ログイン
3. 拡張機能を再試行
```

### 🔧 拡張機能が表示されない

#### 症状
- 右クリックしてもカスタムメニューが表示されない
- 拡張機能アイコンが見つからない

#### 対処法

**1. 拡張機能の有効化確認**
```
手順:
1. chrome://extensions/ を開く
2. 「Google Drive Image Copy」が有効になっているか確認
3. 無効の場合、トグルスイッチをONにする
```

**2. 拡張機能の再読み込み**
```
手順:
1. chrome://extensions/ を開く
2. 「Google Drive Image Copy」の「更新」ボタンをクリック
3. Google Driveのページを再読み込み (F5)
```

**3. アイコンが見つからない場合**
```
手順:
1. Chrome右上のパズルピースアイコンをクリック
2. 「Google Drive Image Copy」を探す
3. ピンアイコンをクリックしてツールバーに固定
```

### ⚠️ エラーメッセージ対応

#### "Extension context invalidated"

**原因**: 拡張機能が更新された際の一時的な問題

```
解決方法:
1. Google Driveページを再読み込み (F5)
2. 問題が続く場合は拡張機能を一度無効化→有効化
```

#### "Failed to execute 'write' on 'Clipboard'"

**原因**: クリップボード書き込み権限がない、またはブラウザの制限

```
解決方法:
1. HTTPSページで実行していることを確認
2. ブラウザのクリップボード設定を確認
3. ブラウザを最新版に更新
```

#### "Tainted canvases may not be exported"

**原因**: CORS制限によりCanvas操作が制限されている

```
これは既知の問題です:
1. 拡張機能が自動的にフォールバック処理を実行
2. 手動コピー指示が表示される場合は指示に従ってください
3. 通常はURL変種の試行により自動解決されます
```

### 🐛 デバッグ方法

#### コンソールでのデバッグ

**1. サービスワーカーのコンソール確認**
```
手順:
1. chrome://extensions/ を開く
2. 「Google Drive Image Copy」の「サービスワーカー」リンクをクリック
3. コンソールでエラーログを確認

期待されるログ:
✅ "Background script loaded"
✅ "Context menu created successfully"
```

**2. ページコンソールの確認**
```
手順:
1. Google Driveページで F12 を押す
2. Console タブを選択
3. 拡張機能のログを確認

期待されるログ:
✅ "Content script loaded on https://drive.google.com/..."
✅ "Right-click detected on element: IMG"
✅ "Found target image element"
```

#### ネットワーク状況の確認

```
手順:
1. F12 → Network タブ
2. 画像コピーを実行
3. 失敗するリクエストがないか確認

確認項目:
- 403 Forbidden: 認証問題
- 404 Not Found: URL問題
- CORS エラー: 制限問題
```

### 🔄 完全リセット手順

問題が解決しない場合の完全リセット:

```
1. 拡張機能の削除
   - chrome://extensions/ で「削除」

2. ブラウザデータのクリア
   - Chrome設定 → プライバシーとセキュリティ
   - 閲覧履歴データの削除
   - 「Cookie と他のサイトデータ」を選択
   - drive.google.com のデータをクリア

3. 拡張機能の再インストール
   - プロジェクトフォルダから再読み込み

4. Google Driveに再ログイン
```

### 📱 ブラウザ固有の問題

#### Chrome

```
既知の問題:
- バージョン 90以下: Clipboard API制限
- シークレットモード: 一部権限制限

解決方法:
- Chrome 91+ を使用
- 通常モードで実行
```

#### Edge

```
既知の問題:
- レガシーEdge: 未対応
- Chromium Edge: Chrome同等

解決方法:
- Chromium Edge (79+) を使用
```

#### Firefox

```
既知の問題:
- Clipboard API実装が部分的
- 一部画像形式で問題

解決方法:
- Firefox 87+ を使用
- Chrome/Edge での使用を推奨
```

### 🔍 FAQ

**Q: すべての画像でコピーできますか？**

A: Google Driveの共有設定により異なります：
- **「リンク共有」設定**: ほぼ100%対応 ✅
- **「一般公開」設定**: 100%対応 ✅
- **「制限付き」設定**: コピー不可 ❌
- **プライベート設定**: コピー不可 ❌

コピーできない場合は、共有設定を「リンク共有」に変更してください。

**Q: 大きな画像もコピーできますか？**

A: はい、ただし制限があります：
- 推奨: 10MB以下
- 最大: ブラウザのメモリ制限まで
- 大きな画像は変換に時間がかかる場合があります

**Q: オフラインでも動作しますか？**

A: いいえ：
- Google Driveへのネットワークアクセスが必要
- 画像データの取得にインターネット接続が必要

**Q: 他のクラウドストレージに対応予定は？**

A: 現在はGoogle Drive専用です：
- Dropbox, OneDriveなどの対応は今後検討
- 技術的な制約により対応困難な場合があります
